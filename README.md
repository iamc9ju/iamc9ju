## เกี่ยวกับผม

ผมให้ความสำคัญกับการออกแบบระบบมากกว่าการเขียนโค้ดให้เสร็จเพียงอย่างเดียว  
สนใจงานที่มีความซับซ้อนเชิงธุรกิจ เช่น ระบบการเงิน ระบบที่มีธุรกรรม และระบบที่ต้องควบคุมสิทธิ์การเข้าถึงข้อมูลอย่างเคร่งครัด

แนวคิดในการทำงาน:

- ออกแบบโครงสร้างให้ชัดเจนก่อนลงมือพัฒนา  
- เขียนโค้ดที่อ่านง่าย ทดสอบได้ และดูแลระยะยาวได้  
- ให้ความสำคัญกับ Transaction Consistency และ Data Integrity  
- มองเรื่อง Security ตั้งแต่ขั้นตอนออกแบบ (Authentication / Authorization / Ownership Validation)

เป้าหมายระยะยาวคือพัฒนาความเชี่ยวชาญด้านการออกแบบระบบระดับองค์กร โดยเฉพาะในสายงานการเงินและธนาคาร

## โครงการที่กำลังพัฒนา

**Banking Core System (Go)**  
ออกแบบระบบบัญชีแบบ Double-entry พร้อม Ledger และ Transaction Processing ที่คำนึงถึง ACID และความถูกต้องของข้อมูลเป็นหลัก

**NutriGo Platform**  
แพลตฟอร์ม Fullstack สำหรับการให้คำปรึกษาด้านโภชนาการ พร้อมระบบสิทธิ์การเข้าถึงข้อมูลและโครงสร้างฐานข้อมูลระดับ Production

**Secure Authorization Module**  
ระบบ RBAC และการป้องกันปัญหา Broken Object Level Authorization ในระดับ API
